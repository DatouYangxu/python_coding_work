***********

正确的Git命令格式是：
git <子命令> [选项] [参数]
例如：
	git status
	git commit -m "message"

***********

如果你想查看所有可用的子命令，可以输入：
 	git help -a
或者查看某个特定子命令的帮助，比如commit：
 	git help commit

***********
Git最常用的10个命令：
1. git init          		初始化												|本地分支名和远程分支名不匹配。比如，本地分支叫master，而远程分支叫main。你可以通过以下命令查看远程分支：
2. git clone         		克隆仓库												|	git remote show origin
3. git add           		添加文件												|或者，你可以列出远程的所有分支：
4. git commit        	提交													|	git ls-remote --heads origin
5. git push          		推送到远程											|如果你本地的分支名是master，而远程期望的是main，你可以：
6. git pull          		从远程拉取											|将本地分支重命名为main：
7. git status        		查看状态												|	git branch -M main
8. git log           		查看历史												|或者，推送时指定分支：
9. git branch        		分支管理												|	git push -u origin master:main   # 将本地的master分支推送到远程的main分支
10. git checkout     	切换分支/恢复文件										|但是，请注意，GitHub现在默认使用main作为默认分支名。
																		|如果你在本地初始化仓库时使用的是master，而远程是main，那么你需要将本地分支名改为main，或者按照远程的分支名																		|来推送。


***********

1. 在任何位置创建新文件夹													|# 基本语法		git pull <远程仓库名> <分支名>
	mkdir 新文件夹名														|# 常用示例		git pull origin main      # 拉取origin仓库的main分支
	cd 新文件夹名	（如：cd /d/桌面（D盘）/my_code_work）				|				git pull origin master    # 如果分支名是master
2. 初始化Git																|# 特殊情况：		
	git init																|第一次拉取		git pull origin main --allow-unrelated-histories
3. 连接到已有的GitHub仓库													|可能需要这个
	git remote add origin https://github.com/你的用户名/仓库名.git			|
4. 拉取远程内容（如果GitHub仓库有文件）									|
	git pull origin main --allow-unrelated-histories							|
5. 然后添加、提交、推送													|
	git add .																|
	git commit -m "说明"													|
	git push -u origin main												|

**********
步骤一：准备本地项目
步骤二：连接GitHub——git remote add origin https://github.com/你的用户名/仓库名.git（添加远程仓库）
步骤三：同步代码 ———git pull origin main --allow-unrelated-histories（先拉取远程的更改（如果GitHub仓库里有文件））
步骤四：提交和推送—— git add . | git commit -m"  " | git push -u origin main
